## 2. 役割分担（重要）

ドメイン購入は **ユーザー本人にしかできない**（支払い・本人情報・ログインが必要）。
Claude側は購入できない。ただしユーザーの作業は「買う」と「言われた値をDNSに貼る」だけ。

- 🧑 ユーザー: お名前.comで購入 / DNSレコードをコピペ
- 🤖 Claude: コード側設定（CNAME・ビルド設定・astro config）/ 貼り付け用DNS値の用意 / 手順実況

> 新Macセッションで「Claude for Chrome」等のブラウザ操作が使える場合でも、
> **カード入力・購入確定はユーザー本人の操作**になる点は変わらない。

---

## 3. これからやる手順

### STEP 1 — ユーザー: お名前.comで購入
1. 検索窓に `shippori-tei`（空きが無ければ `shipporitei`）→ `.com` を選択
2. 申込画面で **レンタルサーバー等のおまけは「申し込まない」**（GitHub Pagesで動くので不要・有料）
3. **Whois情報公開代行は付ける**／**メール転送オプション(有料)は外す**
4. カード登録 → 購入。**自動更新ONのまま**（失効防止）
5. 購入後の勧誘メール/SMSは無視でOK

→ 買えたら **実際に取れたドメイン名** をClaudeに伝える。

### STEP 2 — Claude: サイト(リポジトリ)側の設定
購入ドメインを `DOMAIN`（例: `shippori-tei.com`）とする。

1. **`site/public/CNAME`** を新規作成し、中身を `DOMAIN` の1行だけにする
   （Astroの`public/`はビルドで`dist/`直下にコピーされ、Pagesがカスタムドメインとして認識）
2. **`site/astro.config.mjs`** の `site:` 既定を `https://DOMAIN` に変更
   （10行目 `site: process.env.SITE_URL || 'https://shipporitei.co.jp',`）
3. ビルドのbase-pathについて:
   - 現在 `.github/workflows/pages-preview.yml` は `actions/configure-pages` の出力で
     `BASE_PATH` を注入している。**カスタムドメインをPages設定に登録すると base_path は
     `/`（ルート）になる**ので、基本はそのままで正しく動くはず。
   - 念のためデプロイ後に CSS/画像のリンク切れ(サブパス`/abet-news/`が残ってないか)を確認。
4. ブランチ `claude/shipporitei-official-site-umht7v` にコミット&プッシュ。

### STEP 3 — GitHub Pages側でカスタムドメインを有効化
リポジトリ **Settings → Pages → Custom domain** に `DOMAIN` を入力して保存。
DNS検証が通ったら **Enforce HTTPS にチェック**（証明書は自動発行、数分〜数十分）。

### STEP 4 — ユーザー: お名前.comのDNSにレコードを貼る
お名前.com 管理画面 → **「DNS」→ DNSレコード設定（DNSレコード設定を利用する）**。
Claudeが用意する以下の値を貼る（**GitHub Pagesの固定IP**。2026時点の公式値）。

**Apex（`shippori-tei.com` 自体）に Aレコード4つ:**
```
A  @  185.199.108.153
A  @  185.199.109.153
A  @  185.199.110.153
A  @  185.199.111.153
```
**（任意・IPv6対応するなら AAAA 4つ:）**
```
AAAA  @  2606:50c0:8000::153
AAAA  @  2606:50c0:8001::153
AAAA  @  2606:50c0:8002::153
AAAA  @  2606:50c0:8003::153
```
**`www` を使うなら CNAME:**
```
CNAME  www  j11046rk-commits.github.io.
```
> apexを正規URLにする想定。`www`は任意（GitHubがwww→apexを面倒見る）。
> 反映は最大で数時間。お名前.comのネームサーバーがお名前.com標準であること。

### STEP 5 — 確認
- `https://DOMAIN` でサイトが表示される
- HTTPSの鍵マークが出る（Enforce HTTPS有効）
- 旧URL `https://j11046rk-commits.github.io/abet-news/` からも見られる（併存OK）
